# UX Writing Rule Skill

사용자가 입력한 문구를 토스 UX 라이팅 원칙 8가지 기준으로 분석·검증·개선하는 스킬.

## 원칙 정의 (8가지)

| # | 원칙 | 기준 |
|---|------|------|
| ① | 명확하게 | 유저가 해야 할 액션을 직접 명시했는가? |
| ② | 간결하게 | 불필요한 수식어·중복 표현이 없는가? |
| ③ | 친근하게 | 딱딱한 문어체 지양, 일상 구어체를 사용하는가? |
| ④ | 일관되게 | 동일 개념에 동일 용어를 사용하는가? |
| ⑤ | 단 하나의 핵심만 | 한 문구에 메시지를 하나만 담았는가? 두 가지 이상의 정보를 동시에 전달하지 않는가? |
| ⑥ | 조건과 행동 구체적으로 | "~하면 ~된다" 형식으로 조건(액션)과 결과(혜택)를 구체적으로 명시했는가? |
| ⑦ | 확실한 보상 약속하기 | 리워드·혜택을 모호하지 않고 구체적 수치나 형태로 약속했는가? |
| ⑧ | 자주 쓰는 말로 풀어쓰기 | 전문용어·어려운 표현 없이 누구나 아는 일상 언어로 쓰였는가? |

## 프레이밍 효과 기법 (Framing Techniques)

UX 라이팅 개선 시 아래 프레이밍 기법을 적용한다. (출처: ditoday.com — 전환율을 높여주는 작은 차이)

> **핵심 원칙**: "무엇을 말하느냐보다, 어떻게 말하느냐가 사용자의 경험과 행동에 훨씬 더 큰 영향을 준다."

### 프레이밍 기법 3가지

| # | 기법 | 설명 | Before | After |
|---|------|------|--------|-------|
| F1 | 진입 장벽 해소 (비용 분할) | 큰 비용을 작은 단위로 쪼개 심리적 부담 감소 | "한 달에 3만원으로 무제한 수강" | "매일 1,000원으로 무제한 수강" |
| F2 | 가능성 프레이밍 (제한→기회) | 제한을 알리는 대신 새로운 가능성 제시 | "이 기능은 무료 사용자에게 제공되지 않아요" | "이 기능은 프리미엄에서 이용하실 수 있어요" |
| F3 | 이득 중심 프레이밍 (압박→기회) | 손실 회피 압박 대신 이득·혜택 중심으로 표현 | "타임세일 종료까지 1시간! 절대 놓치지 마세요" | "지금 구매하시면 추가 타임세일을 받으실 수 있어요" |

### 실증 근거

- **구글 (2017)**: "객실 예약" → "예약 가능 여부 확인" = 전환율 **17% 상승** (행동 부담 ↓)
- **국제커뮤니케이션학회 (2018)**: 이득 중심 표현이 손실 회피형 문구보다 **클릭률·전환율 모두 높음**
- **행동경제학**: "95% 안전"이 "5% 위험"보다 훨씬 긍정적으로 인식됨

### 적용 우선순위

Step 2(개선 문구 추천)에서 위반 원칙 수정 시 프레이밍 기법을 함께 적용한다.
Step 4(CTR 최적화)에서 심리 트리거 적용 시, **이득 프레이밍(F3)이 손실 회피보다 효과적**임을 고려한다.

## UX 심리학 법칙 레퍼런스 (Psychology Laws for Writing)

UX 라이팅 개선 시 아래 심리학 법칙을 근거로 활용한다. (출처: brunch.co.kr/@chadwick/33 — 토스에서 찾아본 10가지 UX 심리학 법칙)

### 라이팅 직접 관련 법칙 (7가지)

| # | 법칙 | 핵심 원리 | 연결 원칙 | 라이팅 적용 |
|---|------|----------|----------|------------|
| P1 | 힉의 법칙 (Hick's Law) | 선택지가 많을수록 인지 부하 증가, 의사결정 지연 | ⑤ 단 하나의 핵심만 | 한 문구에 선택지·메시지를 하나만 담아 인지 부하 최소화 |
| P2 | 밀러의 법칙 (Miller's Law) | 작업 기억 용량 7(±2)개 — 정보 덩어리(chunk) 단위 | ② 간결하게 | 긴 문구는 2~3 덩어리로 청킹, 숫자는 010-1234-5678처럼 분절 |
| P3 | 포스텔의 법칙 (Postel's Law) | 사용자 입력은 수용하되, 요청 정보는 최소화 ("1 thing / 1 page") | ⑥ 조건과 행동 구체적으로 | CTA 문구에서 유저에게 요구하는 행동을 1가지로 명확히 제한 |
| P4 | 피크엔드 법칙 (Peak-End Rule) | 경험은 절정·마지막 순간의 감정으로 판단 (실패 경험이 더 오래 기억됨) | ③ 친근하게 | 에러·실패·빈 상태 문구일수록 친근한 톤 필수 (부정 경험 완화) |
| P5 | 폰 레스토프 효과 (Von Restorff) | 비슷한 것들 중 가장 눈에 띄는 것만 기억 | ⑦ 확실한 보상 약속 | 핵심 혜택·리워드 수치를 문구 내에서 가장 두드러지게 배치 |
| P6 | 제이콥의 법칙 (Jacob's Law) | 사용자는 기존 경험(멘탈 모델) 기반으로 새 제품 판단 | ④ 일관되게 | 업계에서 이미 익숙한 용어·패턴 사용 (새로운 조어 지양) |
| P7 | 테슬러의 법칙 (Tesler's Law) | 줄일 수 없는 복잡성이 있으면 시스템이 흡수해야 함 | ⑧ 자주 쓰는 말로 | 복잡한 개념은 시스템이 처리하고, 문구는 결과만 쉽게 전달 |

### 시각/인터랙션 법칙 (참고용 3가지)

| # | 법칙 | 핵심 | 라이팅 간접 영향 |
|---|------|------|-----------------|
| P8 | 피츠의 법칙 (Fitts's Law) | 터치 타겟 크기·거리가 클릭 정확도 결정 | CTA 버튼 문구는 짧고 명확하게 (버튼 크기 제한 고려) |
| P9 | 심미적 사용성 효과 (Aesthetic-Usability) | 예쁜 디자인 = 사용성도 높다고 인식 (50ms 내 판단) | 깔끔한 문구 구조 자체가 신뢰감 형성에 기여 |
| P10 | 도허티 임계 (Doherty Threshold) | 0.4초 내 피드백 필요, 지연 시 로딩 UX로 보상 | 로딩·대기 상태 문구는 진행감 전달 ("확인 중이에요...") |

### 법칙 그룹핑과 원칙 매핑

```
[인지 부하 관리] ← 전환율·완료율 핵심
  P1 힉 + P2 밀러 + P3 포스텔 → ②간결 + ⑤핵심하나 + ⑥조건행동

[심리적 경험] ← 브랜드 신뢰·재사용
  P4 피크엔드 + P6 제이콥 + P9 심미적 → ③친근 + ④일관

[주목·기억] ← 클릭률·행동 유도
  P5 폰레스토프 + P7 테슬러 → ⑦보상약속 + ⑧쉬운말
```

### 적용 우선순위

- Step 1(분석): 원칙 위반 판단 시 **심리학 법칙 근거**를 함께 제시한다 (예: "⑤ 위반 — 힉의 법칙에 따르면 선택지가 2개 이상이면 인지 부하 증가")
- Step 2(개선): 문구 개선 시 관련 법칙을 적용 근거로 설명한다
- Step 4(CTR): 심리 트리거와 심리학 법칙을 조합해 설득력 있는 문구를 생성한다

## Agent Roles

| 에이전트 | 역할 | 출력 |
|---------|------|------|
| **url-context-summarizer** | URL 크롤링 → 제품·타겟·톤 맥락 요약 (URL 입력 시에만 실행) | 구조화된 맥락 요약 블록 (이후 전 Step 프롬프트에 주입) |
| **분석 에이전트** | 입력 문구의 원칙 8가지 준수 여부 판단 | 원칙별 Pass/Fail 결과 |
| **검증 에이전트** | 위반 이유 설명 + 최적 문구 추천 | 개선 문구 1~3개 + 근거 |
| **재검증 에이전트** | 추천 문구의 원칙 재검사 (순차 실행) | 최종 확정 문구 + 통과 리포트 |
| **ctr-hook-writer** | 최종 확정 문구에 심리 트리거 적용 (sub-agent) | 트리거별 CTR 최적 문구 + 최종 추천 1개 |
| **title-description-overlap-checker** | Title·Description 간 어휘·의미·역할 중복 탐지 및 분리 제안 (sub-agent, 쌍 입력 시 실행) | 중복 리포트 + 역할 분리 수정 문구 세트 |

## Step 0: url-context-summarizer Pre-Agent (URL 입력 시에만 실행)

URL이 입력에 포함된 경우 가장 먼저 실행한다.
출력된 맥락 요약 블록은 Step 1~5 모든 프롬프트의 `## 참고 맥락` 섹션에 주입한다.

```
Task(
    subagent_type="general-purpose",
    description="URL 크롤링 및 맥락 요약 (url-context-summarizer)",
    prompt="""
아래 URL의 페이지를 WebFetch로 크롤링하고, UX 라이팅 검증에 활용할 맥락 요약을 만들어줘.

## 대상 URL
[입력 URL]

## 크롤링 추출 항목
1. 페이지 제목 (title 또는 h1)
2. 메타 설명 (meta description)
3. 핵심 본문 (주요 섹션 헤드라인 + 설명)
4. 혜택·리워드 관련 수치 및 표현
5. CTA(Call-to-Action) 버튼 문구
6. 타겟 대상 관련 표현

## 출력 형식 (이후 모든 Step 프롬프트에 주입되는 컨텍스트 블록)

---
## [URL 맥락 요약]
- 제품/서비스명: [식별된 명칭]
- 핵심 가치 제안: [이 제품이 유저에게 주는 핵심 혜택 1~2줄]
- 타겟 유저: [예상 타겟 또는 페이지에서 언급된 대상]

Task(
    subagent_type="general-purpose",
    description="UX 라이팅 원칙 분석",
    prompt="""
아래 문구가 토스 UX 라이팅 원칙 8가지를 준수하는지 분석해줘.

## 검증 문구
[사용자 입력 문구]

## 원칙 기준
① 명확하게: 유저가 취해야 할 액션이 직접적으로 명시되어 있는가?
② 간결하게: 불필요한 수식어, 중복 표현이 없는가?
③ 친근하게: 딱딱한 문어체 대신 구어체를 사용하는가?
④ 일관되게: 동일 개념에 동일 용어를 사용하는가?
⑤ 단 하나의 핵심만: 한 문구에 메시지가 하나만 담겨 있는가? 두 가지 이상의 정보를 동시에 전달하지 않는가?
⑥ 조건과 행동 구체적으로: "~하면 ~된다" 형식으로 조건(액션)과 결과(혜택)가 구체적으로 명시되어 있는가?
⑦ 확실한 보상 약속하기: 리워드·혜택이 모호하지 않고 구체적 수치나 형태로 약속되어 있는가?
⑧ 자주 쓰는 말로 풀어쓰기: 전문용어·어려운 표현 없이 누구나 아는 일상 언어로 쓰였는가?

## 심리학 법칙 참조 (위반 시 근거로 활용)
- P1 힉의 법칙: 선택지 많으면 인지 부하 증가 → ⑤ 관련
- P2 밀러의 법칙: 7±2개 청킹 초과 시 기억 불가 → ② 관련
- P3 포스텔의 법칙: 유저에게 요구하는 행동은 1가지로 제한 → ⑥ 관련
- P4 피크엔드 법칙: 에러·실패 문구일수록 친근한 톤 필수 → ③ 관련
- P5 폰 레스토프 효과: 핵심 혜택이 눈에 띄게 배치되어야 함 → ⑦ 관련
- P6 제이콥의 법칙: 익숙한 용어·패턴 사용해야 함 → ④ 관련
- P7 테슬러의 법칙: 복잡한 개념은 쉽게 풀어 전달 → ⑧ 관련

## 출력 형식
| 원칙 | 결과 | 위반 내용 | 관련 심리학 법칙 |
|------|------|----------|----------------|
| ① 명확하게 | Pass/Fail | ... | — |
| ② 간결하게 | Pass/Fail | ... | P2 밀러 |
| ③ 친근하게 | Pass/Fail | ... | P4 피크엔드 |
| ④ 일관되게 | Pass/Fail | ... | P6 제이콥 |
| ⑤ 단 하나의 핵심만 | Pass/Fail | ... | P1 힉 |
| ⑥ 조건과 행동 구체적으로 | Pass/Fail | ... | P3 포스텔 |
| ⑦ 확실한 보상 약속하기 | Pass/Fail | ... | P5 폰레스토프 |
| ⑧ 자주 쓰는 말로 | Pass/Fail | ... | P7 테슬러 |
"""
)

## Step 4: ctr-hook-writer Sub-Agent (순차 실행, Step 3 완료 후)

Step 3에서 최종 확정된 문구를 받아 마케팅 심리 트리거를 적용한 CTR 최적 문구를 생성한다.

```
Task(
    subagent_type="general-purpose",
    description="CTR 최적화 문구 생성 (ctr-hook-writer)",
    prompt="""
아래 UX 원칙 검증이 완료된 문구에 마케팅 심리 트리거를 적용해줘.

## 최종 확정 문구 (UX 원칙 8가지 통과)
[Step 3 최종 확정 문구]

## 심리 트리거 목록
A. FOMO (기회 손실 공포): 안 하면 놓친다는 불안 자극
B. 긴급성 (Urgency): 즉각 행동 유도
C. 희소성 (Scarcity): 한정 자원 인식
D. 사회적 증명 (Social Proof): 다수의 행동을 따라가려는 심리
E. 손실 회피 (Loss Aversion): 얻는 것보다 잃는 것에 민감한 심리
F. 호기심 격차 (Curiosity Gap): 궁금증 자극으로 클릭 유도
G. 즉시 보상 (Instant Gratification): 즉각적 보상 기대감

## 프레이밍 효과 주의사항 (반드시 참고)
- **이득 프레이밍 > 손실 프레이밍**: 연구에 따르면 "놓치지 마세요" 같은 손실 회피형보다 "지금 하면 혜택을 받을 수 있어요" 같은 이득 중심 표현이 클릭률·전환율 모두 높음
- **진입 장벽 해소**: 큰 비용은 작은 단위로 분할 (월 3만원 → 일 1,000원)
- **가능성 프레이밍**: 제한("안 돼요") 대신 기회("프리미엄에서 가능해요") 표현 사용
- 트리거 E(손실 회피) 적용 시, 공포·압박보다 이득 전환형으로 표현할 것

## 진행 순서
1. 현재 문구에 어떤 트리거가 이미 포함됐는지 분석해줘.
2. 이 문구에 효과적인 트리거 상위 2개를 추천하고 이유를 설명해줘.
3. 추천 트리거 각각을 적용한 문구를 1~2개씩 생성해줘.
   - 단, 토스 UX 원칙(명확하게/간결하게/친근하게/일관되게)은 유지해야 해.
   - 프레이밍 효과 주의사항을 반드시 반영해줘 (이득 프레이밍 우선, 압박 표현 지양).
   - 각 문구에 어떤 심리 원리가 작동하는지 1줄로 설명해줘.
4. 가장 높은 CTR이 예상되는 최종 추천 문구 1개를 선정하고 이유를 설명해줘.

|--------|---------------|------|

- "[문구 2]" → [심리 원리]

**[트리거 B명]**
- "[문구 1]" → [심리 원리]
- "[문구 2]" → [심리 원리]
