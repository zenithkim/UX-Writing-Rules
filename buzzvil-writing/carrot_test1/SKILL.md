# Carrot Test 1 — 꿀피드 C안 템플릿 리라이팅

광고 소재 문구(Title + Description)를 입력하면, 꿀피드 C안 템플릿 기준으로 자동 리라이팅하는 스킬.

## 참조

```
@parent: ~/.claude/skills/buzzvil-writing/SKILL.md (버즈빌 원칙 B1~B4)
@base:   ~/.claude/skills/ux-writing-rule/SKILL.md (UX 원칙 8가지 + 프레이밍 + 심리학)
```

---

## C안 템플릿 정의

### 구조

```
┌─────────────────────────────────────────────┐
│ [아이콘]  [Title]                   [포인트P]│
│           [후킹태그 (파란색)]                 │
└─────────────────────────────────────────────┘
```

### 슬롯 규칙

| 슬롯 | 규칙 | 길이 제한 |
|------|------|----------|
| **Title** | `[브랜드 축약] + [액션]하기` — 브랜드는 핵심 키워드만, 액션은 동사 "~하기"로 통일 | 15자 이내 |
| **후킹태그** | 참여 허들을 낮추거나 보상을 강조하는 짧은 문구 | 6자 이내 |
| **포인트** | 숫자 + "P" 고정 형식 | — |

### 후킹태그 톤 규칙

**통일 톤: 조건 유도형 ("~만하면")**

모든 후킹태그는 "~만하면" 톤으로 통일한다. 유저의 행동 비용을 최소화하는 조건문 형태.

**톤 규칙:**
1. 기본형: `[액션동사]만하면` (6자 이내)
2. 예외 — 시간형: `[시간]이면 끝` / `[시간]면 끝` (소요 시간이 명확한 액션에만 허용)
3. 금지: 명사형("간편 가입"), 동작형("바로 시작"), 형용사형("쉽고 빨리") 사용 금지

### 후킹태그 자동 분류 규칙

입력 문구의 액션 유형에 따라 후킹태그를 자동 선택한다.

| 액션 유형 | 기본 후킹태그 | 시간형 대안 (중복 시) | 비고 |
|----------|-------------|-------------------|------|
| **가입/회원가입** | 가입만하면 | 1분이면 끝 | 동일 브랜드 2개 시 대안 사용 |
| **카드 발급/신청** | 발급만하면 | 5분이면 끝 | 소요시간 명확할 때 시간형 |
| **구독** | 구독만하면 | — | |
| **채널 추가/팔로우** | 터치만하면 | — | 원클릭 액션 통일 |
| **출석/체크인** | 출석만하면 | — | |
| **설치/다운로드** | 설치만하면 | — | |
| **접속/시작** | 접속만하면 | — | |
| **구매/결제/주문** | 구매만하면 / 주문만하면 | — | 첫 주문은 "주문만하면" |
| **조회** | 조회만하면 | — | |
| **확인** | 확인만하면 | — | |
| **등록/프로필** | 등록만하면 | — | |
| **퀴즈** | 풀기만하면 | 30초면 끝 | 초단시간 퀴즈에 시간형 |
| **설문** | 참여만하면 | 1분이면 끝 | |
| **일반 참여** | 참여만하면 | — | |

### 중복 방지 규칙

동일 브랜드 복수 캠페인이 있을 때:
1. 첫 번째 캠페인: 기본 후킹태그 ("~만하면")
2. 두 번째 캠페인: 시간형 대안 ("~이면 끝")
3. 세 번째 이상: Title 동사를 변경하여 후킹태그도 자연스럽게 달라지게

### 브랜드 축약 규칙

| 원본 | 축약 | 규칙 |
|------|------|------|
| "KB국민 노리2 체크카드" | "KB 체크카드" | 카드 상품명 → 은행명 + 카드종류 |
| "쿠팡 로켓그로스 유튜브 채널" | "쿠팡 유튜브" | 서비스명 + 플랫폼 (중간 브랜드 제거) |
| "투썸플레이스 카카오톡 채널" | "투썸 카톡 채널" | 브랜드 줄임말 + 플랫폼 줄임말 |
| "멍과함께 반려견 행복 테스트" | "멍과 함께" | 앱/캠페인명만 남기기 |
| "쿠팡 로켓그로스 신규 가입" | "로켓그로스" | 핵심 서비스명만 |

**축약 원칙:**
1. 유저가 이미 아는 브랜드는 줄임말 사용 (투썸플레이스 → 투썸)
2. 플랫폼은 일상 줄임말 (카카오톡 → 카톡, 유튜브 → 유튜브)
3. 제품 라인업 상세명은 제거 (노리2 → 생략)
4. Title 15자 이내 맞추기 위해 핵심 키워드만 남기기

## Execution Flow

```
┌─────────────────────────────────────────────────┐
│  INPUT: 원본 광고 소재 문구 (1개 또는 여러 개)     │
│  (Title, Desc, 포인트, 과금 모델 등)              │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│  Step 1: 원본 분석                               │
│                                                 │
│  - 브랜드명 식별                                 │
│  - 액션 유형 분류 (가입/발급/구독/출석/...)        │
│  - 과금 모델 식별 (CPC/CPA/CPQ)                  │
│  - 포인트 추출                                   │
│  - AS-IS 문제점 진단 (UX 원칙 + B 원칙 기준)      │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│  Step 2: C안 템플릿 변환                          │
│                                                 │
│  - 브랜드 축약 (축약 규칙 적용)                    │
│  - Title 생성: [브랜드 축약] + [액션]하기          │
│  - 후킹태그 자동 선택 (액션 유형별 분류 규칙)       │
│  - 15자 / 6자 길이 제한 검증                      │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│  Step 3: 원칙 검증                               │
│                                                 │
│  변환 결과를 아래 기준으로 검증:                    │
│  ① UX 원칙 8가지 (@ref ux-writing-rule)          │
│  ② 버즈빌 원칙 B1~B4 (@ref buzzvil-writing)      │
│  ③ C안 슬롯 규칙 (Title 15자, 후킹태그 6자)       │
│  → 미통과 시 Step 2 재실행                        │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│  OUTPUT: C안 리라이팅 결과                        │
│                                                 │
│  | # | Title | 후킹태그 | 포인트 | 변경사항 |      │
│  |---|-------|---------|--------|---------|      │
│  | 1 | ...   | ...     | ...    | ...     |      │
└─────────────────────────────────────────────────┘
```

## Step 프롬프트 상세

### Step 1 + 2 + 3 통합 에이전트

단일 에이전트가 분석 → 변환 → 검증을 순차 실행한다.

```
Task(
    subagent_type="general-purpose",
    description="꿀피드 C안 리라이팅",
    prompt="""
먼저 아래 파일들을 Read하여 지식 베이스를 로드해줘:
1. ~/.claude/skills/ux-writing-rule/SKILL.md (UX 원칙 8가지, 프레이밍 기법, 심리학 법칙)
2. ~/.claude/skills/buzzvil-writing/SKILL.md (버즈빌 원칙 B1~B4, 과금 모델 전략)
3. ~/.claude/skills/buzzvil-writing/carrot_test1/SKILL.md (C안 템플릿 규칙)

## 입력 데이터
[사용자가 입력한 원본 광고 소재 목록]

## 작업 순서

### 1단계: 원본 분석
각 문구에서 아래 요소를 추출해줘:
- 브랜드명 (원본)
- 액션 유형 (가입/발급/구독/출석/설치/구매/참여 중 택1)
- 과금 모델 (CPC/CPA/CPQ, 추정 가능하면)
- 포인트
- AS-IS 문제점 (UX 원칙 + B 원칙 기준, 간단히 1줄)

### 2단계: C안 템플릿 변환
각 문구를 아래 규칙으로 변환해줘:

**Title 규칙:**
- 형식: [브랜드 축약] + [액션]하기
- 브랜드 축약: 유저가 아는 줄임말 사용, 제품 상세명 제거
- 15자 이내 필수

**후킹태그 규칙:**
- 액션 유형에 따라 적합한 유형 자동 선택:
  가입 → 허들 최소화 ("가입만하면", "1분이면 끝")
  발급/신청 → 시간 구체화 ("최대 5분", "바로 발급")
  구독/추가/팔로우 → 난이도 표현 ("쉽고 빨리", "터치 한 번")
  출석/체크인 → 보상 강조 ("연속 보너스", "매일 적립")
  설치 → 허들 최소화 ("무료 설치", "바로 시작")
  구매 → 혜택 강조 ("첫 구매 할인", "추가 적립")
  퀴즈/설문 → 난이도 표현 ("30초면 끝", "쉬운 퀴즈")
- 6자 이내 필수
- 같은 피드 내에서 후킹태그가 겹치지 않도록 변주

### 3단계: 검증
변환 결과가 아래 기준을 모두 통과하는지 확인:
- Title 15자 이내
- 후킹태그 6자 이내
- UX 원칙 ②간결 + ⑤핵심하나 + ⑧쉬운말 통과
- B1 리워드 명확성 (포인트 수치 유지)
- B3 참여 허들 최소화 (후킹태그가 허들을 낮추는가)
- B4 광고 피로도 (피드 내 후킹태그 다양성 확보)

미통과 항목이 있으면 해당 문구만 재변환.

## 출력 형식

### AS-IS → C안 변환 결과

| # | AS-IS Title | AS-IS Desc | → C안 Title | 후킹태그 | 포인트 | 후킹 유형 |
|---|------------|-----------|------------|---------|--------|----------|
| 1 | ... | ... | ... | ... | ...P | 허들최소화/시간구체화/... |

### 변환 근거 요약
각 문구별로 1줄씩:
- #1: [어떤 축약 규칙 적용] + [어떤 후킹태그 유형 선택 이유]

### 검증 결과
- 전체 통과 여부
- 피드 내 후킹태그 다양성 점수 (중복 비율)
"""
)
```
