# Buzzvil Writing Skill

버즈빌 제품의 모든 텍스트(광고 소재, 인앱 메시징, 리워드 문구)를 검증·개선하는 도메인 특화 UX 라이팅 스킬.

## 지식 베이스 참조

이 스킬은 `ux-writing-rule/SKILL.md`를 범용 지식 베이스로 참조한다.
실행 시 아래 경로의 파일을 Read하여 원칙·프레이밍·심리학 법칙을 로드한다.

```
@reference: ~/.claude/skills/ux-writing-rule/SKILL.md
├── 8가지 UX 라이팅 원칙 (What)
├── 프레이밍 효과 기법 3가지 (How)
└── UX 심리학 법칙 10가지 (Why)
```

## 버즈빌 도메인 컨텍스트

### 제품

| 제품 | 설명 |
|------|------|
| **BuzzBenefit** | 리워드 기반 사용자 참여 플랫폼. 광고 참여 시 포인트 적립 |
| **BuzzAd** | 모바일 광고 네트워크. 잠금화면·피드·네이티브 지면 |

### 과금 모델별 문구 전략

| 과금 모델 | 목적 | 문구 전략 | CTA 방향 |
|----------|------|----------|---------|
| **CPC** (클릭당) | 클릭 유도 | 호기심·정보 탐색 욕구 자극 | "자세히 보기", "확인하기" |
| **CPA** (행동당) | 전환 유도 | 혜택 강조 + 행동 명시 | "지금 가입하기", "다운로드" |
| **CPQ** (쿼리당) | 참여 유도 | 낮은 허들 + 즉시 보상 | "참여하기", "응답하기" |

### 핵심 지표 연결

- **R-VER** (반복 가치 경험 도달율): 재방문·재참여를 유도하는 문구
- **ARPDAU**: 수익 전환을 돕는 클릭·참여 문구
- **Click DAU**: 클릭 행동을 이끌어내는 CTA 문구

### 톤앤매너

- 캐주얼하되 신뢰감 있게
- 리워드는 항상 구체적 수치로 ("포인트 적립" ✕ → "100P 적립" ✓)
- 광고주 브랜드를 존중하되, 유저 관점에서 혜택 중심으로

## 버즈빌 전용 원칙 (B 원칙)

기존 8가지 UX 원칙(@reference)에 추가로 적용하는 버즈빌 도메인 원칙.

| # | 원칙 | 기준 | 근거 |
|---|------|------|------|
| **B1** | 리워드 명확성 | 적립 포인트·혜택을 구체적 수치·형태로 명시했는가? | ⑦ 확실한 보상 강화 + 리워드앱 특성 |
| **B2** | 과금 모델 정합성 | 과금 모델(CPC/CPA/CPQ)에 맞는 행동을 유도하는가? | 광고주 ROI 직결 |
| **B3** | 참여 허들 최소화 | "~만 하면" 형태로 행동 비용을 낮게 인식시키는가? | 포스텔의 법칙(P3) + 프레이밍(F1) |
| **B4** | 광고 피로도 고려 | 반복 노출 시 변주 가능한 문구인가? 식상하지 않은가? | 폰 레스토프(P5) 역효과 방지 |

## 문구 유형별 가이드

### A. 광고 소재 카피 (Title + Description)

| 요소 | 권장 길이 | 역할 |
|------|----------|------|
| Title | 15~25자 | 핵심 훅 — 관심 유도, 혜택 1가지 |
| Description | 25~45자 | 보충 맥락 — 조건, 방법, 신뢰 정보 |
| CTA | 2~6자 | 행동 지시 — 과금 모델에 맞는 동사 |

### B. 인앱 메시징

| 유형 | 권장 길이 | 톤 | 예시 |
|------|----------|-----|------|
| 푸시 알림 | ~40자 | 친근 + 긴급/혜택 | "오늘의 미션이 도착했어요" |
| 배너 | ~20자 | 짧고 임팩트 | "지금 참여하면 2배 적립" |
| 툴팁 | ~30자 | 안내형 | "여기를 눌러 포인트를 받아보세요" |
| 바텀시트 | 제목 15자 + 본문 60자 | 설명 + CTA | 제목: "출석체크 보상" / 본문: 상세 |

### C. 리워드 메시징

| 유형 | 권장 길이 | 톤 | 예시 |
|------|----------|-----|------|
| 적립 완료 | ~15자 | 축하, 달성감 | "100P 적립 완료!" |
| 미션 안내 | ~25자 | 격려, 행동 유도 | "영상 보기만 하면 50P 적립" |
| 출석 체크 | ~20자 | 연속감, 기대감 | "3일 연속 출석! 내일은 보너스" |

## 섹션 타이틀 자동 워싱

Unit ID를 입력받아 해당 유닛의 Ad Type Filters 정보를 기반으로 최적의 섹션 타이틀을 자동 생성·워싱하는 기능.

### 플로우

```
INPUT: Unit ID (예: 422455685728323)
    │
    ▼
[Unit 정보 조회]
    │  Unit ID에 포함된 정보 확인:
    │  ① Ad Type Filters → 할당/블락된 adsType(revenue_type) 목록
    │  ② Ad Categories → 광고 카테고리 (보험, 금융, 게임 등)
    │  ③ 최대 리워드 금액 (있으면)
    │
    ▼
[adsType + Ad Categories 기반 섹션 타이틀 생성]
    │
    │  0차: 전체 오퍼월 지면 타이틀 (개인화 프레이밍)
    │  ┌────────────────────────────────────────────────┐
    │  │ "나만을 위한 혜택", "나에게 딱 맞는 미션" 등      │
    │  │  → 지면 전체를 아우르는 상위 타이틀               │
    │  └────────────────────────────────────────────────┘
    │
    │  1차: 카테고리 특화 가능 여부 판단 (섹션 타이틀)
    │  ┌────────────────────────────────────────────────┐
    │  │ 단일/주력 카테고리 식별 가능                      │
    │  │  → 카테고리 + 행동 유도 타이틀                    │
    │  │  → "보험미션 참여하고 포인트 받기"                 │
    │  │                                                │
    │  │ 카테고리 혼합/불특정                              │
    │  │  → adsType 기반 범용 타이틀 생성                  │
    │  └────────────────────────────────────────────────┘
    │
    │  2차: adsType으로 톤 보정
    │  ┌────────────────────────────────────────────────┐
    │  │ CPC 위주  → 탐색·발견 유도형                     │
    │  │ CPM 위주  → 노출·인지 중심형                     │
    │  │ CPA 위주  → 참여·전환 유도형                     │
    │  │ 혼합      → 범용 혜택형                          │
    │  └────────────────────────────────────────────────┘
    │
    ▼
[섹션 타이틀 워싱] (12원칙 적용)
    │
    ▼
OUTPUT: 최적 섹션 타이틀 1~3개 추천
```

### 섹션 타이틀 가이드

| 항목 | 기준 |
|------|------|
| 길이 | 10~20자 |
| 톤 | 캐주얼 + 혜택 중심 |
| 필수 요소 | 유저 행동 유도 or 혜택 암시 |
| 금지 | 광고 느낌 직접 노출 ("광고", "AD"), 과장 표현 |

### 전체 오퍼월 지면 타이틀 (0차 — 지면 레벨)

오퍼월 지면 전체를 아우르는 상위 타이틀. 개인화·큐레이션 느낌을 주어 "광고 목록"이 아닌 "나를 위한 혜택"으로 프레이밍한다.

| 방향 | 예시 |
|------|------|
| **개인화 강조** | "나만을 위한 혜택", "나에게 딱 맞는 미션" |
| **큐레이션 느낌** | "오늘 준비한 혜택", "지금 참여할 수 있는 미션" |
| **발견 유도** | "새로운 혜택이 도착했어요", "놓치기 아쉬운 혜택" |

> **원칙 근거**: 개인화 문구는 제이콥의 법칙(P6 — 넷플릭스·유튜브 등 추천 UI의 멘탈 모델)과 폰 레스토프 효과(P5 — "나만의"가 범용 문구 대비 주목도 ↑)에 기반한다.

### Ad Categories별 타이틀 전략 (1차 — 카테고리 특화 섹션)

카테고리가 단일 또는 주력으로 식별될 때 적용. 금액 표기 없이 카테고리 + 행동 유도 구조.

| Ad Category | 타이틀 방향 | 템플릿 예시 |
|-------------|-----------|-----------|
| **보험** | 보험 미션 + 행동 유도 | "보험미션 참여하고 포인트 받기" |
| **금융** | 금융 혜택 + 간편 참여 | "금융미션 참여하고 포인트 받기" |
| **게임** | 재미 + 보상 | "게임하면서 포인트까지 받기" |
| **쇼핑** | 쇼핑 혜택 + 적립 | "쇼핑하고 포인트 적립하기" |
| **교육** | 학습 + 보상 | "공부하면서 포인트까지 받기" |
| **혼합/불특정** | → 2차 adsType 전략으로 fallback | — |

> **금액 미표기 원칙**: 섹션 타이틀은 하위 광고 묶음의 컨테이너이므로, 특정 금액 약속 시 광고 변동에 따른 기대-현실 괴리 리스크가 있다. 금액은 개별 광고 카피에서 표기한다.

### adsType별 타이틀 전략 (2차 — 범용/톤 보정)

카테고리 특화가 불가능하거나, 1차 결과의 톤을 보정할 때 적용.

| adsType | 유저 기대 | 타이틀 방향 | 예시 |
|---------|----------|-----------|------|
| **CPC** | 클릭해서 정보 탐색 | 호기심·발견 자극 | "놓치면 아쉬운 혜택", "지금 확인해 보세요" |
| **CPM** | 보기만 해도 적립 | 쉬운 참여·낮은 허들 강조 | "보기만 해도 포인트 적립", "쉽게 참여하고 받아가세요" |
| **CPA** | 행동 후 보상 | 참여 유도 + 보상 명시 | "참여하면 바로 적립", "미션 도전하기" |
| **혼합** | 다양한 콘텐츠 | 범용 혜택 프레이밍 | "한눈에 보는 혜택", "오늘의 미션" |

### Step 프롬프트: 섹션 타이틀 워싱

```
Task(
    subagent_type="general-purpose",
    description="섹션 타이틀 자동 워싱",
    prompt="""
먼저 아래 파일을 Read하여 지식 베이스를 로드해줘:
- ~/.claude/skills/ux-writing-rule/SKILL.md (UX 원칙 8가지, 프레이밍 기법, 심리학 법칙)
- ~/.claude/skills/buzzvil-writing/SKILL.md (버즈빌 원칙 B1~B4, 섹션 타이틀 가이드)

## Unit 정보
- Unit ID: [입력된 Unit ID]
- Ad Type Filters: [조회된 adsType 목록]
- 할당된 adsType: [CPC / CPM / CPA / 혼합]
- Ad Categories: [조회된 카테고리 목록 — 보험, 금융, 게임, 쇼핑 등]
- 최대 리워드 금액: [있으면 기입]

## 작업
1. **1차: Ad Categories 분석** — 단일/주력 카테고리가 식별되면 카테고리 특화 타이틀을 생성해줘.
   - 카테고리 특화 시 `{금액}` 플레이스홀더를 최대 리워드 금액으로 치환
   - 예: 보험 카테고리 → "보험미션하고 최대 {금액}원 받아가세요!"
2. **2차: adsType으로 톤 보정** — 카테고리 혼합/불특정이면 adsType 기반 범용 타이틀을 생성해줘.
3. 해당 방향으로 섹션 타이틀 후보 3개를 생성해줘.
   - 10~20자 길이
   - 12원칙(UX 8 + B1~B4) 준수
   - 프레이밍 기법(F1~F3) 적용 가능하면 적용
4. 각 후보를 12원칙으로 검증하고 최종 추천 1개를 선정해줘.

## 출력 형식

### Unit 분석
| 항목 | 값 | 타이틀 영향 |
|------|-----|-----------|
| adsType | ... | ... |
| Ad Categories | ... | ... |
| 최대 리워드 | ... | ... |
| **전략 판단** | 카테고리 특화 / adsType 범용 | ... |

### 섹션 타이틀 후보
| # | 타이틀 | 글자수 | 적용 전략 | 적용 원칙/기법 |
|---|--------|--------|----------|--------------|
| 1 | ... | ...자 | ... | ... |
| 2 | ... | ...자 | ... | ... |
| 3 | ... | ...자 | ... | ... |

### 최종 추천
> "[타이틀]"
> 추천 이유: [선정 근거]
"""
)
```

## Step 프롬프트 상세

### Step 0: 문구 유형 분류

사용자가 과금 모델이나 문구 유형을 명시하지 않으면 AskUserQuestion으로 확인한다.

### Step 1: 통합 원칙 분석

```
Task(
    subagent_type="general-purpose",
    description="버즈빌 UX 라이팅 통합 분석",
    prompt="""
먼저 아래 파일을 Read하여 UX 라이팅 지식 베이스를 로드해줘:
- ~/.claude/skills/ux-writing-rule/SKILL.md (8가지 원칙, 프레이밍 기법, 심리학 법칙)

그 후 아래 문구를 12가지 원칙(UX 8가지 + 버즈빌 4가지)으로 분석해줘.

## 검증 문구
[사용자 입력 문구]

## 문구 유형: [광고 소재 / 인앱 메시징 / 리워드]
## 과금 모델: [CPC / CPA / CPQ / 해당없음]

## UX 원칙 (ux-writing-rule 참조)
① 명확하게 / ② 간결하게 / ③ 친근하게 / ④ 일관되게
⑤ 단 하나의 핵심만 / ⑥ 조건과 행동 구체적으로 / ⑦ 확실한 보상 약속 / ⑧ 자주 쓰는 말로

## 버즈빌 원칙
B1 리워드 명확성: 포인트·혜택을 구체적 수치로 명시했는가?
B2 과금 모델 정합성: 과금 모델에 맞는 행동을 유도하는가?
B3 참여 허들 최소화: "~만 하면" 형태로 행동 비용을 낮게 인식시키는가?
B4 광고 피로도 고려: 반복 노출 시 변주 가능한가?

## 출력 형식
| 원칙 | 결과 | 위반 내용 | 관련 심리학 법칙 |
|------|------|----------|----------------|
| ① ~ ⑧ | Pass/Fail | ... | P1~P7 |
| B1~B4 | Pass/Fail | ... | 근거 |
"""
)
```

### Step 2: 개선 문구 추천

```
Task(
    subagent_type="general-purpose",
    description="버즈빌 문구 개선 추천",
    prompt="""
아래 분석 결과를 바탕으로 개선 문구를 추천해줘.

## 원본 문구
[사용자 입력 문구]

## 문구 유형: [유형] / 과금 모델: [모델]

## 분석 결과
[Step 1 결과]

## 프레이밍 기법 (ux-writing-rule 참조)
- F1 진입 장벽 해소 (비용 분할)
- F2 가능성 프레이밍 (제한→기회)
- F3 이득 중심 프레이밍 (압박→기회)

## 요청
1. 각 위반 원칙에 대해 이유를 설명하고, 관련 심리학 법칙(P1~P7)을 근거로 제시해줘.
2. 12가지 원칙 모두 준수하는 최적 문구를 1~3개 추천해줘.
   - 프레이밍 기법(F1~F3) 적용 가능하면 함께 적용
   - 과금 모델 전략에 맞는 CTA 방향 반영
3. 각 추천 문구가 어떤 원칙·기법·법칙을 적용했는지 설명해줘.
"""
)
```

### Step 3: 재검증

Step 2 추천 문구를 12가지 원칙(8+4)으로 재검증. 통과 시 확정, 미통과 시 Step 2 재실행.

### Step 4: CTR 최적화

@reference ctr-hook-writer 서브 에이전트 실행. 과금 모델별 트리거 우선순위 적용.

### Step 5: Title-Description 중복 체크

광고 소재(Title+Desc) 입력 시에만 @reference title-description-overlap-checker 실행.
